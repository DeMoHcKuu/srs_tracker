Техническое задание на проект: SRS-Study Tracker

1) Цель проекта
Создать веб-сервис интервального повторения (SRS), который помогает пользователю учить материал карточками
и автоматически планирует дату следующего повторения на основе оценок качества ответа (0–5) по алгоритму SM-2.

2) Роли пользователей
- Гость: может видеть лендинг/описание сервиса, страницу демо (без сохранения прогресса).
- Пользователь (авторизованный): создает колоды и карточки, проходит повторения, смотрит аналитику прогресса.
- Администратор: управляет данными через Django admin (карточки/колоды/теги), модерирует контент при необходимости.

3) Модели данных (сущности)
1) Deck (Колода)
- user (FK -> User)
- title (str)
- description (text)
- created_at (datetime)

2) Card (Карточка)
- deck (FK -> Deck)
- front_text (text) — вопрос/термин
- back_text (text) — ответ/объяснение
- created_at (datetime)
- is_active (bool)

3) Review (Повторение/результат)
- user (FK -> User)
- card (FK -> Card)
- reviewed_at (datetime) — когда повторили
- quality (int 0..5) — оценка ответа
- interval_days (int) — рассчитанный интервал
- ease_factor (float) — коэффициент “легкости”
- next_review_at (date/datetime) — когда повторить снова

4) Ключевой функционал (User Stories)
- Пользователь регистрируется/входит, создает колоду и добавляет карточки через формы.
- Пользователь открывает страницу “Повторить сегодня” и получает список карточек, где next_review_at <= сегодня.
- Пользователь оценивает ответ (0–5), система сохраняет Review и пересчитывает next_review_at, interval_days, ease_factor.
- Пользователь открывает страницу аналитики и видит:
  - график количества повторений по дням,
  - график среднего quality по дням/неделям,
  - таблицу “самые сложные карточки” (низкий average quality / высокий % ошибок).

5) Внешние интеграции / Аналитика
- Аналитика: расчёт расписания повторений по SM-2 + агрегирующие запросы (Count/Avg) для статистики.
- Визуализация: Plotly (интерактивные графики на странице аналитики).
